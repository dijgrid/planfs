# Backlog

The backlog view is for work that is not fully shaped yet. Use it to capture new task ideas, filter by refinement state, and open the structured editor when an item needs more detail.

Good backlog review usually answers three questions:

- Is the task clear enough for someone else to pick up?
- Does it have the priority, owner, dates, and relationships needed for planning?
- Are unresolved questions visible instead of hidden in conversation?

The list, filters, grouping, and editor panel are all working against the Markdown task files in `.planfs/tasks`.
